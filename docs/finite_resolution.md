# FRPM in MoeaBench: FAIR Metrics (Physical Layer) and Q-Scores (Clinical Layer)

This document describes the **analysis framework** implemented in MoeaBench to audit MOEA outcomes when both the **Ground Truth (GT)** and the **returned front** are **finite, discrete point sets**. The framework is organized as two layers:

- **FAIR metrics (Physical layer):** quantities grounded in geometry and probability that remain in “error units” (typically *lower is better*), designed to be interpretable and comparable across finite-sampling regimes.
- **Q-scores (Clinical layer):** normalized scores in $[0, 1]$ (*higher is better*) obtained by mapping FAIR values to *interpretable quality levels* using **statistical baselines**.

The emphasis is on **auditability** and **finite-resolution realism**. The goal is not to replace classical MOEA indicators, but to provide an additional layer that tends to remain meaningful when the GT is discrete and when the algorithm output has finite cardinality.

---

## 1. Introduction

### 1.1 Why a “finite-resolution” framework?
In most MOEA benchmarks, the “true Pareto front” is not an analytic surface in the evaluation pipeline. Instead, it is a **finite reference set** $R\subset\mathbb{R}^M$ sampled from that surface (or estimated empirically). Likewise, the algorithm output is another **finite set** $P\subset\mathbb{R}^M$, typically of size $K\le 200$.

When both $R$ and $P$ are finite:
- Some “perfect” targets (e.g., a literal zero distance to a continuous manifold) can be **physically unattainable** in practice.
- Comparisons can become **sensitive** to the density of $R$ and to the size $K$ of $P$, especially as $M$ grows.

MoeaBench addresses this by separating:
1. **what is measured** (FAIR = physical facts), from
2. **how those facts are interpreted** (Q = clinical judgement anchored by baselines).

### 1.2 The two-layer logic at a glance
Given a front $P$ and a reference $R$, MoeaBench computes a set of FAIR metrics:

- `HEADWAY` and `CLOSENESS` (progress and proximity, in *resolution units*),
- `COVERAGE` and `GAP` (extent and continuity, in *objective-space distance*),
- `REGULARITY` and `BALANCE` (distribution geometry and regional parity).

Then each FAIR metric is mapped to a Q-score:

- `Q_HEADWAY`, `Q_CLOSENESS`, `Q_COVERAGE`, `Q_GAP`, `Q_REGULARITY`, `Q_BALANCE`.

The Q-score layer uses baselines that attempt to reflect:
- an **ideal** (or “best plausible”) finite-sampling behavior for the given problem and $K$, and
- a **worst-case / random** behavior under an explicit null model.

In general, this makes the audit more stable than relying on a single absolute threshold that does not adapt to the finite-resolution regime.

## 3. Evaluation model: two finite point sets in objective space

### 3.1 Objects, notation, and assumptions
Let:
- $R = \{r_i\}_{i=1}^{M_R}$ be the GT/reference set (finite and discrete),
- $P = \{p_j\}_{j=1}^{K}$ be the algorithm output front (finite and discrete),
- the objective space be $\mathbb{R}^M$, with $M$ objectives.

MoeaBench treats both $R$ and $P$ as point clouds in the same objective space.

A core primitive used repeatedly is the distance from a point to a set:
$$
d(x, S) = \min_{s\in S} \|x - s\|_2.
$$

From that, two directional set-to-set distance families arise naturally:
- **P→R**: how far the produced points are from the reference manifold,
- **R→P**: how far the reference manifold is from the produced approximation (coverage).

### 3.2 K as a first-class quantity (and why MoeaBench “snaps” it)
Many baselines depend explicitly on $K$. To keep the clinical mapping consistent, MoeaBench applies a **K-target policy**:

- If $K < 10$, it is treated as $10$ (minimum supported for baselines).
- If $10 \le K < 50$, it uses the exact $K$.
- If $K \ge 50$, it snaps to the largest value in $\{50, 100, 150, 200\}$ that is $\le K$.

This is implemented by `baselines.snap_k(K_raw)` and is used to select baseline distributions and reference constructs consistently across runs.

### 3.3 Deterministic reference constructs derived from GT
Several FAIR metrics and baselines require deterministic “reference objects” derived from $R$, in particular:

- A deterministic **Uniform-at-K** reference subset $U_K$, generated by **Farthest Point Sampling (FPS)** on $R$:
  $$
  U_K = \mathrm{FPS}(R, K; \text{seed}=0).
  $$
  This is used as a canonical finite-sampling “best-coverage” pattern at size $K$.

- Deterministic **cluster centroids** (e.g., via k-means on $R$ with fixed seed) used for occupancy/parity analyses in `BALANCE`.

These deterministic objects allow the framework to use “ideal” references that are reproducible and auditable.

## 4. Finite resolution and the role of the resolution factor $s_K$

### 4.1 Why a resolution factor at all?
When $R$ is dense, its internal nearest-neighbor distances can be extremely small (especially in higher dimensions). If one normalizes by the *GT’s* microscopic spacing, many practical distances become huge in normalized units, and downstream scores can lose discriminative power.

MoeaBench therefore uses a **macroscopic ruler** aligned with the finite sampling regime at size $K$.

### 4.2 Definition of $s_K$ used by MoeaBench
MoeaBench defines a resolution factor $s_K$ as the **median nearest-neighbor spacing** inside $U_K$:

1. Build $U_K = \mathrm{FPS}(R, K; \text{seed}=0)$.
2. For each $u\in U_K$, compute its nearest neighbor distance within $U_K$:
   $$
   \mathrm{NN}(u) = \min_{v\in U_K, v\ne u} \|u-v\|_2.
   $$
3. Define:
   $$
   s_K = \mathrm{median}\big(\{\mathrm{NN}(u)\}_{u\in U_K}\big).
   $$

Interpretation: $s_K$ is a *typical spacing* expected for a set of $K$ points that are “as uniformly spread as possible” along the discrete reference set. This makes $s_K$ a natural ruler for measuring convergence in **finite-resolution units**.

### 4.3 Where $s_K$ is used
In the current MoeaBench diagnostics pipeline, $s_K$ plays a central role in:
- `HEADWAY` (and thus `Q_HEADWAY`),
- `CLOSENESS` (and thus `Q_CLOSENESS`),
and it is also used when generating baseline distributions (notably the **GT-blur** baseline for closeness).

Other FAIR metrics (coverage, gap, regularity, balance) are defined directly in objective-space terms or in probability/divergence terms, and are interpreted clinically through their own baselines.

## 6. FAIR metrics (Physical layer): definitions, intuition, implementation
FAIR metrics are “physical facts.” They are intentionally **not** mapped into $[0,1]$ here. In general:
- **Lower** FAIR values represent “less pathology” (closer to ideal physical behavior).
- Each metric targets a distinct geometric or distributional property.

All FAIR metrics are implemented in `moeabench.diagnostics.fair`.

### 6.1 FAIR `HEADWAY` — progress toward the manifold (resolution units)
**Purpose.** Quantify how far the population still is from the GT manifold, expressed in units of the finite-resolution ruler $s_K$.

**Definition.**
For each $p_j\in P$, compute $d(p_j,R)$. Let $\{d_j\}$ be these distances. `HEADWAY` uses a robust high-percentile:
$$
GD_{95}(P\to R) = \mathrm{percentile}_{95}(\{d_j\}).
$$
Then normalize:
$$
\mathrm{HEADWAY}(P,R) = \frac{GD_{95}(P\to R)}{s_K}.
$$

**Why the 95th percentile?**
The 95th percentile tends to reflect the “bulk” of the population while still being sensitive to a non-trivial tail. It often helps avoid a single outlier dominating the conclusion, while still penalizing broadly shifted populations.

**Implementation notes.**
- If $|R|$ is large, MoeaBench uses a KDTree for the nearest-neighbor queries.
- The function returns a `FairResult` with:
  - `.value` = the scalar headway,
  - `.raw_data` = the distribution of normalized point-to-GT distances (useful for plotting and additional diagnostics).

**Interpretation.**
- Values near 0 indicate the population is physically very close to the GT manifold.
- Values $\gg 1$ indicate the population is many “finite-resolution units” away, which tends to be visually apparent as a shifted or remote front.

---

### 6.2 FAIR `CLOSENESS` — median proximity to the manifold (resolution units)
**Purpose.** Describe the typical (central) proximity of solutions to the GT manifold in resolution units.

**Definition.**
Compute normalized point-to-manifold distances:
$$
u_j = \frac{d(p_j,R)}{s_K}.
$$
Then:
$$
\mathrm{CLOSENESS}(P,R) = \mathrm{median}(\{u_j\}).
$$

**Why the median?**
The median is robust to tails and is intended to represent “typical closeness.” In practice, it complements `HEADWAY`, which is more tail-sensitive.

**Implementation notes.**
- Like `HEADWAY`, it uses KDTree when helpful.
- It returns `FairResult` with `.raw_data = \{u_j\}` so the full distribution is available.

**Interpretation.**
- Small values mean the population is tightly concentrated near the manifold.
- Larger values suggest “blurry” convergence: the population may be near the right region but still spread away from the manifold.

---

### 6.3 FAIR `COVERAGE` — average extent of approximation (objective-space distance)
**Purpose.** Measure how well the produced front spans the full GT manifold.

**Definition.**
For each reference point $r_i \in R$, compute distance to the nearest produced point:
$$
\delta_i = d(r_i,P) = \min_{p\in P}\|r_i - p\|_2.
$$
Then:
$$
\mathrm{COVERAGE}(P,R) = \mathrm{mean}(\{\delta_i\}).
$$
This is essentially an IGD-mean-style extent measure (direction $R \to P$).

**Implementation notes.**
- KDTree is used when $|P|$ is large.
- Returned `raw_data` contains the $\{\delta_i\}$ values over the GT.

**Interpretation.**
- Smaller values generally indicate a front that “covers” the reference manifold more completely.
- In finite sampling regimes, this value typically does not reach zero unless $P$ matches $R$ very closely everywhere.

---

### 6.4 FAIR `GAP` — robust worst-case coverage hole (objective-space distance)
**Purpose.** Detect large holes and fragmentation in the approximation.

**Definition.**
Using the same $\delta_i = d(r_i,P)$ distances as in coverage:
$$
\mathrm{GAP}(P,R) = \mathrm{percentile}_{95}(\{\delta_i\}).
$$

**Why a 95th percentile “max”?**
A strict max can be dominated by a tiny number of extreme reference points or numerical artifacts. The 95th percentile is intended to reflect “large holes that matter” while remaining somewhat robust.

**Interpretation.**
- A low gap suggests the manifold is covered continuously without severe holes.
- A high gap suggests significant missing regions (fragmentation).

---

### 6.5 FAIR `REGULARITY` — spacing order relative to a uniform-at-K reference
**Purpose.** Measure whether the spacing among points in $P$ resembles a well-ordered “uniform” spacing pattern at size $K$.

**Definition.**
Let $NN(P)$ be the empirical distribution of nearest-neighbor distances within $P$ (excluding self):
$$
NN_P(j) = \min_{l\ne j}\|p_j - p_l\|_2.
$$
Let $U_K$ be the uniform-at-K reference subset derived from $R$, and similarly define $NN(U_K)$.
Then `REGULARITY` is the 1D Wasserstein distance between these two distributions:
$$
\mathrm{REGULARITY}(P,R) = W_1\big(NN(P), NN(U_K)\big).
$$

**Why Wasserstein on NN distances?**
Nearest-neighbor spacing captures local “order” vs “clumping.” Wasserstein distance provides a stable way to compare one-dimensional distributions (spacing patterns), often more informative than comparing only means/variances.

**Interpretation.**
- Values near 0 indicate spacing patterns close to the reference uniform lattice.
- Larger values indicate irregular spacing, clumping, or noisy scatter.

---

### 6.6 FAIR `BALANCE` — regional parity of mass over the manifold
**Purpose.** Measure whether the produced points are distributed evenly across “regions” of the GT manifold.

**Definition.**
- Fit a fixed set of centroids (e.g., k-means on $R$, with deterministic seed), obtaining $C = \{c_m\}$.
- Assign each $p\in P$ to its nearest centroid, producing an occupancy histogram $H_P$.
- Compute the same histogram for $U_K$, yielding $H_{ref}$.
Then:
$$
\mathrm{BALANCE}(P,R) = D_{JS}\big(H_P \,\|\, H_{ref}\big),
$$
where $D_{JS}$ is Jensen–Shannon divergence (base 2), which lies in $[0,1]$.

**Interpretation.**
- Values near 0 indicate parity close to the reference occupancy.
- Larger values indicate bias: an over-concentration in some regions and under-coverage in others.

## 7. Q-scores (Clinical layer): per-metric mappings and baseline rationale
Q-scores map FAIR values into a $[0,1]$ quality axis:
- $Q \approx 1$: behavior close to the **ideal baseline**,
- $Q \approx 0$: behavior at or worse than the **worst-case/random baseline**.

In MoeaBench, baselines are stored per triple:
$$
(\text{problem}, K, \text{metric}),
$$
and include:
- `uni50`: a median “ideal” baseline value,
- `rand50`: a median “worst-case/random” baseline value,
- `rand_ecdf`: 200 sorted samples defining an ECDF-based baseline distribution.

Baselines are accessed through `moeabench.diagnostics.baselines`.

### 7.1 Baseline design (ideal vs worst-case/random)

#### 7.1.1 Ideal baseline: “Uniform-at-K” draws
For most metrics, MoeaBench defines an *ideal* as something like:
- “a deterministic or near-deterministic uniform subset of the manifold at the same $K$.”

Concretely, in baseline calibration, MoeaBench generates repeated “ideal draws” via FPS-derived $U_K$ with varying seeds, and takes the median (`uni50`) as the anchor.

This is intended to represent **best plausible finite-sampling behavior** rather than an unattainable continuous optimum.

#### 7.1.2 Worst-case/random baseline: explicit null models
MoeaBench adopts pessimistic null models depending on the dimension being assessed:

- For `COVERAGE`, `GAP`, `REGULARITY`, and `BALANCE`, the random baseline typically comes from sampling points uniformly in the objective-space box $[0,1]^M$ (a bounding-box null), then computing the FAIR metrics against the GT-derived references. This aims to approximate a “no-search” behavior in objective space.

- For `CLOSENESS`, the random baseline is not bounding-box random. Instead, it uses a **GT-blur** model: points are displaced from the manifold along estimated normal directions with a calibrated noise level. This baseline is designed to produce a “plausible near-manifold but still incorrect” proximity distribution, which can improve resolution when distinguishing “sharp convergence” from “blurry convergence.”

- For `HEADWAY`, the clinical anchor uses:
  - **ideal** = 0 (no distance to the manifold),
  - **random** = headway distribution under the bounding-box null.

#### 7.1.3 Why use medians (`uni50`, `rand50`)?
Using medians tends to provide a stable “typical” anchor in the presence of skewed distributions. This is especially relevant when the random baseline distribution has a long tail. In addition, the ECDF mapping uses the full baseline distribution, so the median acts mainly as a robust quality threshold.

---

### 7.2 Q-score mapping functions used by MoeaBench
MoeaBench uses three mapping styles, depending on the metric.

#### 7.2.1 ECDF-based mapping (coverage, gap, regularity, balance)
For many FAIR metrics, MoeaBench maps the measured FAIR value $f$ using the ECDF of the random baseline:

Let $F_{rand}$ be the ECDF derived from `rand_ecdf` (200 samples). Define:
- `ideal` = `uni50`,
- `rand50` = the median of the random baseline,
and compute:
$$
Q(f) = 1 - \mathrm{clip}_{[0,1]}\left(\frac{F_{rand}(f) - F_{rand}(\mathrm{ideal})}{F_{rand}(\mathrm{rand50}) - F_{rand}(\mathrm{ideal})}\right).
$$

Intuition:
- When $f$ is as good as the ideal anchor, $F_{rand}(f)$ is near $F_{rand}(\mathrm{ideal})$, hence $Q\approx 1$.
- When $f$ reaches the random median, $Q\approx 0$.
- The ECDF provides a probability-space normalization that can be less sensitive than raw linear scaling when baseline distributions are skewed.

#### 7.2.2 Log-linear mapping (headway)
For headway, MoeaBench uses a log-linear transform:
$$
Q(f) = 1 - \mathrm{clip}_{[0,1]}\left(\frac{\log(1 + (f - \mathrm{ideal}))}{\log(1 + (\mathrm{rand50} - \mathrm{ideal}))}\right),
$$
with $\mathrm{ideal}=0$.

Intuition:
- Headway values can span a wider range; a log mapping can preserve resolution near the top end (near-ideal progress) without collapsing all good solutions into indistinguishable scores.

#### 7.2.3 Distributional mapping (closeness)
For closeness, MoeaBench maps entire distributions rather than a single scalar.
Let:
- $U$ be the empirical distribution of normalized distances $u_j = d(p_j,R)/s_K$,
- $I$ be the ideal distribution (all zeros),
- $R_{blur}$ be the baseline distribution stored as `rand_ecdf` under `closeness`.

Compute Wasserstein distances:
- $d_I = W_1(U, I)$,
- $\Delta = W_1(R_{blur}, I)$,
- $d_R = W_1(U, R_{blur})$.

Then MoeaBench computes a ratio form:
- If $d_I \ge \Delta$, the score is set to 0 (worse-than-baseline gate).
- Otherwise:
  $$
  Q = \frac{d_R}{d_R + d_I}.
  $$

Intuition:
- $Q\approx 1$ if the front is close to the ideal distribution.
- $Q\approx 0$ if the front resembles (or is worse than) the blurred baseline.
- Using distributions allows the score to reflect “sharpness” of convergence, not only a single median.

---

### 7.3 The Q-scores, one by one

#### 7.3.1 `Q_HEADWAY`
- **Input FAIR metric:** `HEADWAY`.
- **Ideal anchor:** $0$.
- **Worst-case/random anchor:** `rand50` from the headway baseline distribution (bounding-box null).
- **Mapping:** log-linear.
- **Meaning:** how much progress was made toward the manifold beyond what would be expected under a no-search (random box) behavior.

#### 7.3.2 `Q_CLOSENESS`
- **Input FAIR metric:** full distribution of normalized distances from `CLOSENESS.raw_data`.
- **Ideal anchor:** all zeros (perfect sharp convergence).
- **Worst-case/random anchor:** GT-blur distribution (`closeness.rand_ecdf`).
- **Mapping:** Wasserstein distributional ratio with a monotonicity gate.
- **Meaning:** sharpness and precision of convergence, beyond merely “being in the right region.”

#### 7.3.3 `Q_COVERAGE`
- **Input FAIR metric:** `COVERAGE`.
- **Ideal anchor:** `uni50` (median of “uniform-at-K” ideal draws).
- **Worst-case/random anchor:** `rand50` (median under bounding-box random).
- **Mapping:** ECDF.
- **Meaning:** whether the front spans the manifold broadly, given finite $K$.

#### 7.3.4 `Q_GAP`
- **Input FAIR metric:** `GAP`.
- **Ideal anchor:** `uni50`.
- **Worst-case/random anchor:** `rand50`.
- **Mapping:** ECDF.
- **Meaning:** continuity of coverage (absence of large holes), anchored by the random baseline.

#### 7.3.5 `Q_REGULARITY`
- **Input FAIR metric:** `REGULARITY`.
- **Ideal anchor:** `uni50` (often near 0 if the ideal draw matches the reference distribution).
- **Worst-case/random anchor:** `rand50`.
- **Mapping:** ECDF.
- **Meaning:** whether the spacing pattern resembles an ordered uniform-at-K pattern.

#### 7.3.6 `Q_BALANCE`
- **Input FAIR metric:** `BALANCE`.
- **Ideal anchor:** `uni50` (often near 0).
- **Worst-case/random anchor:** `rand50`.
- **Mapping:** ECDF.
- **Meaning:** whether mass is distributed equitably across manifold regions, compared to the reference occupancy.

## 8. Reference implementation in MoeaBench

### 8.1 Public entry points
MoeaBench exposes three “cascade” entry points in `moeabench.diagnostics.auditor`:

- `fair_audit(target, ground_truth=GT, ...)`
  Returns the **physical** FAIR metrics.

- `q_audit(target, ground_truth=GT, ...)`
  Returns the **clinical** Q-scores.

- `audit(target, ground_truth=GT, ...)`
  Runs both layers and returns a combined “biopsy” report with a synthesis verdict.

### 8.2 What can be passed as `target`
The diagnostics context resolver accepts:
- a raw numpy array of objective points,
- a Population-like object with `.objectives`,
- an Experiment/Run-like object with `.front()`.

When passing raw arrays, it is recommended to also provide:
- `ground_truth=...` and
- `problem="DTLZ2"` (or similar),
so baseline selection can be performed deterministically.

### 8.3 Baseline loading and fail-closed policy
Baselines are stored offline in a JSON file and are loaded through:
- `baselines.load_offline_baselines()`

If required baselines are missing or malformed (e.g., ECDF length not equal to 200, or not sorted), the pipeline is designed to behave conservatively and raise an `UndefinedBaselineError` rather than silently “guessing.”

For research workflows, this fail-closed behavior tends to be preferable because it prevents accidental grade inflation caused by unintended fallback logic.

### 8.4 Calibrating baselines for new problems (sidecar model)
For custom MOPs, MoeaBench supports generating a sidecar calibration JSON:
- `mop.calibrate(...)`

Calibration builds, for each $K$ in a grid:
- `uni50` from repeated ideal (uniform-at-K) draws,
- `rand50` and `rand_ecdf` from repeated worst-case/random draws,
- a dedicated closeness blur baseline (`closeness.rand_ecdf`).

The sidecar can be registered via `baselines.register_baselines(path_or_dict)`, enabling clinical scoring for new problems without modifying the core baseline resource.

### 8.5 Performance considerations (nearest-neighbor queries and caching)
Many FAIR metrics require repeated nearest-neighbor queries. MoeaBench uses:
- KDTree acceleration when the reference set is large,
- caching of KDTree objects and FPS results to avoid repeated expensive constructions during calibration or repeated audits.

## 9. Practical usage patterns and interpretation

### 9.1 Typical audit workflow
A common pattern is to:
1. compute FAIR metrics and inspect them as physical evidence,
2. compute Q-scores to obtain normalized judgements,
3. read the synthesis narrative as a compact explanation.

For example (illustrative):

```python
import moeabench as mb

# P: array of objective points from your MOEA
# GT: reference point set (analytical PF, external GT, or sidecar GT)
report = mb.diagnostics.audit(P, ground_truth=GT, problem="DTLZ2")

print(report.report(markdown=True))
print(report.summary())  # narrative synthesis.
```
